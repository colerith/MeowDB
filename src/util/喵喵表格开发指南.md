# MeowDB - 喵喵表格插件开发指南（整合版）

## 1. 项目定位

| 项目 | 说明 |
|------|------|
| 名称 | MeowDB / 喵喵表格 |
| 类型 | SillyTavern 插件（Extension） |
| 核心功能 | 基于 FM 模板的剧情数据管理与可视化 |
| 数据存储 | 聊天消息内嵌（`mes.data.meowDB`），按聊天隔离 |
| 技术栈 | Vite + Vue3 + Pinia + TypeScript |
| 构建产物 | `dist/index.js` + `dist/index.css` |

---

## 2. 数据模型

### 2.1 MeowDBEntry（当前实现）

```ts
interface MeowDBEntry {
  serial: string;
  time: string;
  nsfw: {
    current: number;
    max: number;
    details?: Record<string, unknown>;
  };
  scene: {
    main: string;
    sub: string;
    stayRounds: number;
    topic: string;
  };
  plot: string;
  relations: CharacterRelation[];
  echoes: Echo[];
  archived: ArchivedNPC[];
  enigmas: Enigma[];
  seeds: Seed[];
}
```

### 2.2 存储约定

- 读取：从聊天末尾向前查找最新 `chat[i].data.meowDB`
- 写入：保存到最新消息 `lastMsg.data.meowDB`
- 持久化：调用 `SillyTavern.saveChat()`

## 4. UI 注入规范

### 4.1 顶栏配置入口

- 注入位置：`#top-settings-holder`
- 排序策略：强制插在最后一个图标前
- 图标风格：与酒馆一致的 `drawer-icon + fa-fw + FontAwesome`
- 交互方式：点击图标打开独立 Modal（不依赖 `drawer-content`）

### 4.2 可视化面板

- 注入位置：`#chat` 内，紧跟 `.mes.last_mes` 后
- 层级要求：与消息同级，不挂到设置区
- 目标效果：消息区内卡片式可视化面板

## 9. UI 设计基线（新增）

为避免被 SillyTavern 全局样式污染，后续 UI 必须遵循以下规则：

### 9.1 样式作用域

- 所有视觉样式必须限定在 `.meowdb-modal` 与 `.meowdb-visual-shell` 内。
- 禁止编写无作用域的 `h1/h2/h3/strong/::before/::after` 全局样式。
- 标题默认伪元素清空（防酒馆主题注入），仅允许组件类（如 `.meowdb-setting-title`）主动定义 before/after。

### 9.2 字体与层级

- 统一 UI 字体栈：`PingFang SC / Microsoft YaHei UI / Noto Sans SC / Segoe UI`。
- 统一代码字体栈：`JetBrains Mono / Consolas`。
- 标题层级建议：`h1 24` / `h2 20` / `h3 18`；`strong` 使用 760~780 字重。
- 文案基调：高对比深蓝灰，不使用纯黑。

### 9.3 组件语言（当前风格）

- 卡片：浅玻璃背景 + 细边框 + 中等圆角（12~16）。
- 间距：8/10/12 为主，不做大留白。
- 按钮：统一高度与对齐；功能按钮可压缩但字重清晰。
- 复选框：仅使用 MeowDB 自定义图标，不依赖宿主默认伪元素。
- 滚动条：仅在插件容器内自定义，保持细轨道与圆角滑块。

### 9.4 交互提示

- 新建/重命名等输入动作优先使用内嵌小弹层，不再使用 `prompt()`。
- 导入/导出/删除必须有即时 toast 反馈（成功/失败/原因）。
- 复杂设置先“选择方案”再“编辑字段”，减少重复输入。

### 9.5 后续模块复用要求

- 新模块落地前，先复用现有变量：字体、圆角、边框、按钮、滚动条。
- 视觉组件优先复用：`meowdb-settings-section`、`meowdb-setting-title`、`meowdb-tool-btn`、`meowdb-input`。
- 若新增视觉 token，先写入 `src/global.css` 或 `src/styles/meowdb-theme.css` 的 MeowDB 作用域区，再使用。

## 10. UI Token 与覆盖策略（细化）

### 10.1 样式文件分层

- `src/global.css`：功能样式与历史兼容样式。
- `src/styles/meowdb-theme.css`：UI token、字体守卫、按钮/滚动条强覆盖。
- `src/index.ts`：必须按顺序引入 `global.css` -> `meowdb-theme.css`，确保主题覆盖生效。

### 10.2 必须使用的视觉 token

- 字体：`--meowdb-font-ui` / `--meowdb-font-mono`
- 圆角：`--meowdb-radius-sm/md/lg`
- 颜色：`--meowdb-text` / `--meowdb-text-muted` / `--meowdb-accent`
- 边框：`--meowdb-border` / `--meowdb-border-strong`
- 滚动条：`--meowdb-scroll-track` / `--meowdb-scroll-thumb`

后续模块禁止直接写硬编码色值，优先使用 token。

### 10.3 标题/强调文本规则

- 插件内 `h1/h2/h3/strong` 必须在作用域内重置，禁止依赖酒馆默认样式。
- 非 MeowDB 组件标题默认清空伪元素；需要装饰线时仅在组件类（如 `.meowdb-setting-title`）里定义。

### 10.4 控件与按钮规则

- 点状图标按钮（关闭/切换）统一最小尺寸 `34x34`。
- 以下两个节点必须走单独样式：
  - `#chat > div.meow-db-panel > section > aside > article > button`
  - `#MeowDB_settings_overlay > div > button`
- hover/active 统一：轻抬升 + 阴影增强 + 点击回弹。

### 10.5 复选框规则

- 仅使用 `.meowdb-check`，并强制屏蔽宿主 `::before`。
- 勾选图标统一使用 FontAwesome 字形（非手绘 path）。
- 若主题切换后位置偏移，优先调整 `translate`，不要改容器尺寸。

### 10.6 滚动条强覆盖策略

- Modal、可视化、`#MeowDB_settings_overlay`、`#chat .meow-db-panel` 全部使用 `!important` 的滚动条规则。
- 同时覆盖 `scrollbar-width` 与 `::-webkit-scrollbar*`，兼容 Chromium/Firefox。
- 轨道和滑块必须保持圆角，hover 颜色仅做轻微加深。
