# MeowDB - 喵喵表格插件开发指南（整合版）

## 1. 项目定位

| 项目 | 说明 |
|------|------|
| 名称 | MeowDB / 喵喵表格 |
| 类型 | SillyTavern 插件（Extension） |
| 核心功能 | 基于 FM 模板的剧情数据管理与可视化 |
| 数据存储 | 聊天消息内嵌（`mes.data.meowDB`），按聊天隔离 |
| 技术栈 | Vite + Vue3 + Pinia + TypeScript |
| 构建产物 | `dist/index.js` + `dist/index.css` |

---

## 2. 数据模型

### 2.1 MeowDBEntry（当前实现）

```ts
interface MeowDBEntry {
  serial: string;
  time: string;
  nsfw: {
    current: number;
    max: number;
    details?: Record<string, unknown>;
  };
  scene: {
    main: string;
    sub: string;
    stayRounds: number;
    topic: string;
  };
  plot: string;
  relations: CharacterRelation[];
  echoes: Echo[];
  archived: ArchivedNPC[];
  enigmas: Enigma[];
  seeds: Seed[];
}
```

### 2.2 存储约定

- 读取：从聊天末尾向前查找最新 `chat[i].data.meowDB`
- 写入：保存到最新消息 `lastMsg.data.meowDB`
- 持久化：调用 `SillyTavern.saveChat()`

## 4. UI 注入规范

### 4.1 顶栏配置入口

- 注入位置：`#top-settings-holder`
- 排序策略：强制插在最后一个图标前
- 图标风格：与酒馆一致的 `drawer-icon + fa-fw + FontAwesome`
- 交互方式：点击图标打开独立 Modal（不依赖 `drawer-content`）

### 4.2 可视化面板

- 注入位置：`#chat` 内，紧跟 `.mes.last_mes` 后
- 层级要求：与消息同级，不挂到设置区
- 目标效果：消息区内卡片式可视化面板
